<Info>
  **Scope required:** `account:read`
</Info>

Returns the calling workspace's identity, active plan, and subscription state.

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="workspace" type="string">
      The workspace slug.
    </ResponseField>

    <ResponseField name="name" type="string">
      The workspace's display name.
    </ResponseField>

    <ResponseField name="plan" type="object">
      <Expandable title="properties">
        <ResponseField name="slug" type="string">
          The plan's slug.
        </ResponseField>

        <ResponseField name="tier" type="string">
          The plan's tier.
        </ResponseField>

        <ResponseField name="name" type="string">
          The plan's display name.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="subscription" type="object">
      <Expandable title="properties">
        <ResponseField name="status" type="string">
          The subscription status.
        </ResponseField>

        <ResponseField name="valid" type="boolean">
          Whether the subscription is currently valid.
        </ResponseField>

        <ResponseField name="ends_at" type="string">
          When the current subscription period ends.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request GET \
  --url https://api.whatsmark.io/api/v1/account \
  --header "Authorization: Bearer YOUR_API_TOKEN"
```

</RequestExample>

Common error responses:

| Status | error\_code | When it happens |
| --- | --- | --- |
| 401 | `unauthenticated` | Missing, invalid or expired API token. |
| 402 | `subscription_required` | The workspace has no valid subscription, or a payment is under review. |
| 403 | `insufficient_scope` | The token lacks the required scope, or the account is suspended/on hold. |
| 429 | `quota_exhausted` | The monthly API request quota for the workspace is exhausted. |
