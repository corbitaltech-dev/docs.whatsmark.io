<Info>
  **Scope required:** `contacts:write`
</Info>

Create a contact. `phone` is required and must be unique in E.164-ish form. Honours the `contacts` plan limit, returning `422 limit_reached` at cap.

<ParamField body="first_name" type="string" required>
  The contact's first name. Max length 100.
</ParamField>

<ParamField body="last_name" type="string">
  The contact's last name. Max length 100.
</ParamField>

<ParamField body="phone" type="string" required>
  Must be unique within your workspace. Max length 20.
</ParamField>

<ParamField body="email" type="string">
  The contact's email address. Max length 255.
</ParamField>

<ParamField body="type" type="string">
  One of `guest`, `lead`, or `customer`.
</ParamField>

<ParamField body="contact_status_id" type="integer">
  Must reference an existing `contact_statuses` record.
</ParamField>

<ParamField body="contact_source_id" type="integer">
  Must reference an existing `contact_sources` record.
</ParamField>

<ParamField body="is_active" type="boolean">
  Whether the contact is active.
</ParamField>

<ParamField body="group_ids" type="integer[]">
  Each id must reference an existing `contact_groups` record.
</ParamField>

<ParamField body="label_ids" type="integer[]">
  Each id must reference an existing `contact_labels` record.
</ParamField>

<ParamField body="custom_fields" type="object">
  Arbitrary custom field values, e.g. `company`.
</ParamField>

<ParamField body="display_name" type="string">
  The contact's display name. Max length 255.
</ParamField>

<ResponseField name="data" type="object">
  The created contact.

</ResponseField>

<RequestExample>

```bash Request
curl --request POST \
  --url https://api.whatsmark.io/api/v1/contacts \
  --header "Authorization: Bearer YOUR_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "first_name": "Jane",
    "last_name": "Doe",
    "phone": "+919876543210",
    "email": "jane@example.com",
    "type": "lead",
    "contact_status_id": 1,
    "contact_source_id": 1,
    "is_active": true,
    "group_ids": [1],
    "label_ids": [1],
    "custom_fields": {"company": "Acme"}
  }'
```

</RequestExample>

Errors: `401 unauthenticated` for a missing/invalid token, `402 subscription_required` when the workspace has no valid subscription, `403 insufficient_scope` when the token lacks `contacts:write`, `422 limit_reached` when the contacts plan limit is hit, and `429 quota_exhausted` when the monthly request quota is exhausted.
