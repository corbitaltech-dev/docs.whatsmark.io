<Info>
  **Scope required:** `channels:read`
</Info>

Returns all connected messaging accounts for the workspace across **every platform** (WhatsApp, Messenger, Instagram, Telegram, …), each with their endpoints nested under `endpoints`. All accounts include `id`, `platform`, `name`, `account_id`, `status`, `connection_method`, `is_default`, `connected_at` and `endpoints`. WhatsApp accounts additionally include the phone-number shape (`waba_id`, `business_profile`, `mm_lite_status`, `phone_numbers`) for backward compatibility. The default channel is listed first.

Error responses use a shared `Error` shape (`message`, `error_code`, `status`). Common cases: `401` when the API token is missing, invalid or expired (`unauthenticated`); `402` when the workspace has no valid subscription or a payment is under review (`subscription_required`); `403` when the token lacks the `channels:read` scope or the account is suspended/on hold (`insufficient_scope`); `429` when the monthly API request quota is exhausted (`quota_exhausted`).
