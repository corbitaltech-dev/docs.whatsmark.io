<Info>
  **Scope required:** `channels:read`
</Info>

Returns a single connected account by its ID on any platform, with its endpoints nested under `endpoints`. WhatsApp accounts additionally include the phone-number shape (`waba_id`, `phone_numbers`, …) for backward compatibility. Returns `404` if the channel does not exist or belongs to a different workspace.

<ParamField path="channel_id" type="string" required>
  Identifier of the channel.
</ParamField>

Error responses use a shared `Error` shape (`message`, `error_code`, `status`). Common cases: `401` when the API token is missing, invalid or expired (`unauthenticated`); `402` when the workspace has no valid subscription or a payment is under review (`subscription_required`); `403` when the token lacks the `channels:read` scope or the account is suspended/on hold (`insufficient_scope`); `404` when the channel does not exist or belongs to a different workspace (`not_found`); `429` when the monthly API request quota is exhausted (`quota_exhausted`).
