<Info>
  **Scope required:** `otp:send`
</Info>

Send a one-time passcode over WhatsApp using an APPROVED **authentication**-category template. The 6-digit code is generated server-side, hashed at rest, and delivered via the template's body \+ URL button. Returns `202` with an opaque `reference`.

Rate limits: 5/hour and 20/day per phone; 60s resend cooldown. `channel_id` must be a WhatsApp phone-number (endpoint) channel; `template_id` must belong to its account and be authentication-category \+ APPROVED.

<ParamField body="phone" type="string" required>
  The recipient's phone number, in E.164-like format. Maximum length 20 characters.
</ParamField>

<ParamField body="channel_id" type="integer" required>
  Must reference an existing `channels` record.
</ParamField>

<ParamField body="template_id" type="integer" required>
  Must reference an existing `message_templates` record.
</ParamField>

<ParamField body="purpose" type="string">
  Optional free-form purpose label for this OTP. Maximum length 50 characters.
</ParamField>

<ParamField body="expiry_minutes" type="integer">
  Optional expiry, in minutes. Must be between 1 and 60.
</ParamField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="reference" type="string">
      Opaque identifier for this OTP, used to verify, cancel, resend, or check status.
    </ResponseField>

    <ResponseField name="phone" type="string">
      The recipient's phone number.
    </ResponseField>

    <ResponseField name="channel_id" type="integer">
      The channel the OTP was sent from.
    </ResponseField>

    <ResponseField name="purpose" type="string">
      The purpose label for this OTP.
    </ResponseField>

    <ResponseField name="status" type="string">
      The OTP's current status, e.g. `pending`.
    </ResponseField>

    <ResponseField name="expires_at" type="string">
      Timestamp when the OTP expires.
    </ResponseField>

    <ResponseField name="verified_at" type="string">
      Timestamp when the OTP was verified, or `null` if not yet verified.
    </ResponseField>

    <ResponseField name="verification_attempts" type="integer">
      Number of verification attempts made so far.
    </ResponseField>

    <ResponseField name="remaining_attempts" type="integer">
      Number of verification attempts remaining.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Timestamp when the OTP was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request POST \
  --url https://api.whatsmark.io/api/v1/otp \
  --header "Authorization: Bearer YOUR_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "phone": "+919876543210",
    "channel_id": 2,
    "template_id": 1,
    "purpose": "login",
    "expiry_minutes": 10
  }'
```

| error\_code | when |
| --- | --- |
| `unauthenticated` (401) | Missing, invalid or expired API token. |
| `subscription_required` (402) | The workspace has no valid subscription, or a payment is under review. |
| `insufficient_scope` (403) | The token lacks the `otp:send` scope, or the account is suspended/on hold. |
| `invalid_template` (422) | Template not found, not approved, or not an authentication category. |
| `rate_limit_exceeded` (429) | Hourly OTP limit reached for this number. |
| `send_rate_limited` (429) | Send rate limit reached for this WhatsApp number (includes a `Retry-After` header). |
