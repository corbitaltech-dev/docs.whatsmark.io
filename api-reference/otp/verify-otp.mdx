api: "POST /api/v1/otp/{reference}/verify"

title: "Verify OTP"
description: "Submit the code the user received to verify an OTP."
---

<Info>
  **Scope required:** `otp:verify`
</Info>

Submit the code the user received. On success `status` becomes `verified`. Wrong codes decrement `remaining_attempts` (`invalid_code`); other failures: `otp_expired`, `already_verified`, `max_attempts_reached`, `otp_cancelled`.

<ParamField path="reference" type="string" required>
  Identifier of the reference.
</ParamField>

<ParamField body="code" type="string" required>
  The code to verify. Must be between 4 and 8 characters.
</ParamField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="reference" type="string">
      Opaque identifier for this OTP.
    </ResponseField>

    <ResponseField name="phone" type="string">
      The recipient's phone number.
    </ResponseField>

    <ResponseField name="channel_id" type="integer">
      The channel the OTP was sent from.
    </ResponseField>

    <ResponseField name="purpose" type="string">
      The purpose label for this OTP.
    </ResponseField>

    <ResponseField name="status" type="string">
      The OTP's current status, e.g. `verified`.
    </ResponseField>

    <ResponseField name="expires_at" type="string">
      Timestamp when the OTP expires.
    </ResponseField>

    <ResponseField name="verified_at" type="string">
      Timestamp when the OTP was verified.
    </ResponseField>

    <ResponseField name="verification_attempts" type="integer">
      Number of verification attempts made so far.
    </ResponseField>

    <ResponseField name="remaining_attempts" type="integer">
      Number of verification attempts remaining.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Timestamp when the OTP was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request POST \
  --url https://api.whatsmark.io/api/v1/otp/9b1f8c2a-4e7d-4a1b-9c3e-2f5a6b7c8d9e/verify \
  --header "Authorization: Bearer YOUR_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "code": "123456"
  }'
```

</RequestExample>

| error\_code | when |
| --- | --- |
| `unauthenticated` (401) | Missing, invalid or expired API token. |
| `subscription_required` (402) | The workspace has no valid subscription, or a payment is under review. |
| `insufficient_scope` (403) | The token lacks the `otp:verify` scope, or the account is suspended/on hold. |
| `invalid_code` (422) | Invalid code; `remaining_attempts` is decremented. |
| `quota_exhausted` (429) | The monthly API request quota for the workspace is exhausted. |

Other possible `422` failures for this endpoint: `otp_expired`, `already_verified`, `max_attempts_reached`, `otp_cancelled`.
