---
api: "POST /api/v1/statuses"
playground: "simple"
title: "Create Status"
description: "Creates a new contact status with a unique name and sort order."
---

<Info>
  **Scope required:** `statuses:write`
</Info>

Create a status. `name` and `sort_order` must be unique.

<ParamField body="name" type="string" required>
  Must be unique within your workspace. Maximum length 100.
</ParamField>

<ParamField body="color" type="string">
  Hex color for the status. Maximum length 7. Nullable.
</ParamField>

<ParamField body="sort_order" type="integer">
  Must be unique within your workspace. Nullable.
</ParamField>

<ParamField body="is_default" type="boolean">
  Whether this status should be the default. Nullable.
</ParamField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Unique identifier of the status.
    </ResponseField>

    <ResponseField name="name" type="string">
      Name of the status.
    </ResponseField>

    <ResponseField name="color" type="string">
      Hex color of the status.
    </ResponseField>

    <ResponseField name="sort_order" type="integer">
      Position of the status relative to other statuses.
    </ResponseField>

    <ResponseField name="is_default" type="boolean">
      Whether this is the default status.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Timestamp when the status was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request POST \
  --url https://api.whatsmark.io/api/v1/statuses \
  --header "Authorization: Bearer YOUR_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "name": "VIP",
    "color": "#FF0000",
    "sort_order": 1,
    "is_default": false
  }'
```

</RequestExample>

<ResponseExample>

```json 201 Created
{
  "data": {
    "id": 1,
    "name": "VIP",
    "color": "#FF0000",
    "sort_order": 1,
    "is_default": false,
    "created_at": "2026-06-26T10:00:00+00:00"
  }
}
```

</ResponseExample>

Errors: `401` (missing/invalid token), `402` (no valid subscription), `403` (missing `statuses:write` scope), `422` (validation failed, e.g. `name` missing or not unique), `429` (quota exhausted).
