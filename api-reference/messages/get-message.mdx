---
api: "GET /api/v1/messages/{message_id}"
playground: "simple"
title: "Get Message"
description: "Fetch a single message with its delivery status."
---

<Info>
  **Scope required:** `messages:read`
</Info>

Fetch a single message with its delivery status (`queued` → `sent` → `delivered` → `read`, or `failed`).

<ParamField path="message_id" type="string" required>
  Identifier of the message.
</ParamField>

<ResponseField name="data" type="object">
  The message.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      The message identifier.
    </ResponseField>

    <ResponseField name="conversation_id" type="integer">
      The conversation this message belongs to.
    </ResponseField>

    <ResponseField name="contact_id" type="integer">
      The recipient contact identifier.
    </ResponseField>

    <ResponseField name="channel_id" type="integer">
      The channel the message was sent through.
    </ResponseField>

    <ResponseField name="type" type="string">
      The message type, e.g. `template`.
    </ResponseField>

    <ResponseField name="status" type="string">
      The message delivery status, e.g. `delivered`.
    </ResponseField>

    <ResponseField name="body">
      The text body, if applicable.
    </ResponseField>

    <ResponseField name="metadata">
      Additional message metadata.
    </ResponseField>

    <ResponseField name="error_message">
      Error message, if the send failed.
    </ResponseField>

    <ResponseField name="sent_at" type="string">
      Timestamp the message was sent.
    </ResponseField>

    <ResponseField name="delivered_at" type="string">
      Timestamp the message was delivered.
    </ResponseField>

    <ResponseField name="read_at">
      Timestamp the message was read.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Timestamp the message record was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request GET \
  --url https://api.whatsmark.io/api/v1/messages/88 \
  --header "Authorization: Bearer YOUR_API_TOKEN"
```

</RequestExample>

Common error responses: `401` when the API token is missing, invalid, or expired; `402` when the workspace has no valid subscription; `403` when the token lacks the `messages:read` scope; `404` when the message does not exist; `429` when the monthly API request quota is exhausted (includes `RateLimit-Limit`, `RateLimit-Remaining`, and `RateLimit-Reset` headers).
