---

<Info>
  **Scope required:** `messages:send`
</Info>

Convenience endpoint to send a template to a specific contact by path ID. The request body omits `contact_id` and `type`, since both are implied. Sends are throttled to Meta's per-number rate limit.

<ParamField path="contact_id" type="string" required>
  Identifier of the contact.
</ParamField>

<ParamField body="channel_id" type="integer">
  Must reference an existing `channels` record.
</ParamField>

<ParamField body="template_id" type="integer">
  Identifier of the approved template to send.
</ParamField>

<ParamField body="template_data" type="object">
  Template variable data.

  <Expandable title="properties">
    <ParamField body="components" type="object[]">
      Array of template components used to fill template variables.

      <Expandable title="properties">
        <ParamField body="type" type="string">
          The component type, e.g. `body`.
        </ParamField>

        <ParamField body="parameters" type="object[]">
          Array of parameter values for the component.

          <Expandable title="properties">
            <ParamField body="type" type="string">
              The parameter type, e.g. `text`.
            </ParamField>

            <ParamField body="text" type="string">
              The parameter text value.
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="status" type="string">
  The overall response status, e.g. `queued`.
</ResponseField>

<ResponseField name="data" type="object">
  The created message.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      The message identifier.
    </ResponseField>

    <ResponseField name="conversation_id" type="integer">
      The conversation this message belongs to.
    </ResponseField>

    <ResponseField name="contact_id" type="integer">
      The recipient contact identifier.
    </ResponseField>

    <ResponseField name="channel_id" type="integer">
      The channel the message was sent through.
    </ResponseField>

    <ResponseField name="type" type="string">
      The message type, e.g. `template`.
    </ResponseField>

    <ResponseField name="status" type="string">
      The message delivery status, e.g. `queued`.
    </ResponseField>

    <ResponseField name="body">
      The text body, if applicable.
    </ResponseField>

    <ResponseField name="metadata">
      Additional message metadata.
    </ResponseField>

    <ResponseField name="error_message">
      Error message, if the send failed.
    </ResponseField>

    <ResponseField name="sent_at">
      Timestamp the message was sent.
    </ResponseField>

    <ResponseField name="delivered_at">
      Timestamp the message was delivered.
    </ResponseField>

    <ResponseField name="read_at">
      Timestamp the message was read.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      Timestamp the message record was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Request
curl --request POST \
  --url https://api.whatsmark.io/api/v1/contacts/1/messages \
  --header "Authorization: Bearer YOUR_API_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "channel_id": 2,
    "template_id": 1,
    "template_data": {
      "components": [
        {
          "type": "body",
          "parameters": [
            { "type": "text", "text": "#A123" }
          ]
        }
      ]
    }
  }'
```

</RequestExample>

Common error responses: `401` when the API token is missing, invalid, or expired; `402` when the workspace has no valid subscription; `403` when the token lacks the `messages:send` scope; `422 no_phone` when the contact has no phone number; `429 send_rate_limited` when Meta's per-number send rate is exceeded (includes a `Retry-After` header).
