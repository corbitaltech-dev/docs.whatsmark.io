If you're bringing contacts over from another system, you don't need to add them one at a time — use the **Import** wizard from the Contacts page.

## Importing contacts

<Steps>
<Step title="Upload your file">
  From Contacts, click **Import Contacts**, then **New Import**, and upload a **CSV or XLSX** file (up to 30 MB).

  <BrowserFrame
    src="/contacts/img/import-upload.png"
    url="whatsmark.io/contacts/import"
    alt="Upload step of the import wizard with a Before you upload checklist and a drag-and-drop file area accepting CSV or XLSX up to 30 MB"
    caption="Image shows the upload step with the before-you-upload checklist and file drop area."
  />

  Your file needs a **header row** as the first row, and phone numbers must include the **country code** (for example `+919876543210`). Duplicate phone numbers are **skipped** — existing contacts are never overwritten.
</Step>

<Step title="Map your columns">
  WhatsMark.io reads your file's headers and suggests a mapping to contact fields automatically. Review and adjust the mapping — including your own [custom fields](/core-setup/custom-fields) — before continuing.

  <BrowserFrame
    src="/contacts/img/import-map-columns.png"
    url="whatsmark.io/contacts/import"
    alt="Map Columns step showing a preview of the first rows of the uploaded file above dropdowns matching each file column to a contact field, with Phone and First Name marked required"
    caption="Image shows the Map Columns step with a file preview and field mapping dropdowns."
  />
</Step>

<Step title="Choose defaults and start the import">
  Set a default **status**, **source**, **group(s)**, **label(s)**, **WhatsApp channel**, and **assigned team member** applied to every contact in this file (used when a row doesn't specify its own). Choose **Start Import** to queue it as a background job, so large files don't block you from continuing to use the app.

  <BrowserFrame
    src="/contacts/img/import-default-values.png"
    url="whatsmark.io/contacts/import"
    alt="Default Values step showing an import summary of the column mapping, a note confirming how many rows were detected and how many contact slots remain, and default fields for channel, type, status, source, assignee, groups, and labels"
    caption="Image shows the Default Values step with the import summary and default field settings."
  />
</Step>
</Steps>

<Note>
Some defaults are **required** — the WhatsApp channel, type, status, and source must be set before you can start the import.
</Note>

## Tracking an import

<BrowserFrame
  src="/contacts/img/import-history.png"
  url="whatsmark.io/contacts/import"
  alt="Import Contacts page listing past imports with the file name, a progress bar showing rows processed with counts of imported and skipped rows, a Completed status, and the date"
  caption="Image shows the Import Contacts page listing past imports with their progress."
/>

Each import you run is listed with its status and progress. Open one to:

- See how many rows succeeded and how many failed.
- **Download an error report** listing exactly which rows failed and why.
- **Download the original file** you uploaded.
- **Delete** a single import record, or **bulk-delete** several at once. This only removes the import history entry — contacts already created by that import are not deleted.

<Note>
A failed row doesn't stop the rest of the import — WhatsMark.io processes every row it can and reports failures separately, so one bad phone number won't block hundreds of good ones.
</Note>

## Exporting contacts

From the Contacts page, apply whatever filters you want (status, source, group, label, and so on), then use **Export** to download exactly that filtered list as a file.

<Tip>
Export respects your current filters — export a specific group or label if you only need a subset, rather than exporting everything and filtering afterward.
</Tip>
