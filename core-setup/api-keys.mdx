If you want to integrate with WhatsMark.io programmatically — your own app, an external system, a custom automation — you'll need workspace **API tokens**.

<Warning>
API access is a **Professional** plan feature — Free-plan workspaces can't create tokens or reach the API host until they upgrade. Every plan with API access also has a monthly request quota that scales with your plan.
</Warning>

## Creating a token

<Steps>
<Step title="Click Create Token">
  Give your token a descriptive name so you remember what it's for later (e.g., "Mobile App", "Zapier Integration", "Scheduled Backup").
</Step>

<Step title="Choose abilities (scopes)">
  Select exactly which parts of the API this token can access — account, channels, contacts, groups, labels, sources, statuses, messages, templates, conversations, OTP, and usage are all separately scoped. Grant only the minimum permissions your integration needs.
</Step>

<Step title="Set an expiry (optional)">
  Tokens can be created with or without an expiration date. Expiring tokens add an extra security layer by forcing periodic rotation.
</Step>

<Step title="Copy your token">
  The full token value is shown **once**, right after creation — copy it immediately and store it securely. WhatsMark.io only stores a hash of it afterward and can't show it to you again.
</Step>
</Steps>

<Warning>
If you lose a token's value, you can't retrieve it — you'll need to **regenerate** it (which invalidates the old value but keeps the same name and abilities) or create a new one.
</Warning>

## Managing tokens

**View token details:**
- See when each token was **last used** to spot ones that are no longer active
- Check which **scopes** (abilities) each token has
- View the token's **creation date** and **expiry date** (if set)

**Token actions:**
- **Regenerate** a token's secret in place, keeping its name and abilities intact (the old token is invalidated immediately)
- **Delete** a token to revoke access immediately
- **Extend expiry** (if the token is set to expire)

<Tip>
Regularly review your tokens and delete any that are no longer in use. For integrations that need frequent rotation, set an expiry date and regenerate before expiration.
</Tip>

## Using your token

Every request to the API needs your token as a bearer token in the Authorization header:

```text
Authorization: Bearer YOUR_API_TOKEN
```

**Example curl request:**
```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
  https://api.whatsmark.io/api/v1/contacts
```

See the full [API Overview](/overview) for the base URL, pagination, endpoint reference, and error handling.
