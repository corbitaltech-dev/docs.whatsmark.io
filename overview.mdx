## What is the WhatsMark API

The WhatsMark REST API lets you interact with your workspace programmatically — contacts, templates, messages, conversations, channels, and OTP verification — instead of using the dashboard by hand. It's served from a dedicated host, `api.whatsmark.io`, separate from your tenant subdomain.

<Warning>
  **The API is not available on the Free plan.** REST API access is a **Professional** plan feature — Free-plan workspaces can use the full dashboard, but any request to `api.whatsmark.io` will be rejected until you upgrade. Reseller plans also include API access, with a higher (or unlimited) monthly request quota.
</Warning>

## Getting access

<Steps>
  <Step title="Upgrade to Professional">
    If you're on the Free plan, upgrade from **Billing** in the account menu (click your name at the bottom of the sidebar) first. See [Manage your plan and billing](/getting-started#manage-your-plan-and-billing).
  </Step>
  <Step title="Create a workspace API token">
    Go to **Setup → API Keys** and create a token. Choose the abilities (scopes) it needs — tokens are only granted the scopes you select.
  </Step>
  <Step title="Authenticate your requests">
    Every request must include your token as a bearer token:

    ```text
    Authorization: Bearer YOUR_API_TOKEN
    ```
  </Step>
  <Step title="Call the API">
    Send requests to `https://api.whatsmark.io/api/v1/...`. Start with `GET /api/v1/ping` to confirm your token and abilities are working.
  </Step>
</Steps>

## Base URL

```text
https://api.whatsmark.io/api/v1
```

## Scopes (abilities)

Each endpoint requires a specific token ability (scope), for example `account:read` or `channels:read`. Create your token with the abilities that match the endpoints you plan to call — a request will be rejected with `403 Forbidden` if your token is missing the required scope.

## Pagination

List endpoints return a consistent envelope:

```json
{
  "data": [ ... ],
  "links": { "next": "...", "prev": "..." },
  "meta": { ... }
}
```

Large or append-only collections (contacts, group members, webhooks) use opaque cursor pagination — keep following `links.next` until it's `null`. `per_page` is clamped between 5 and 100.

## Async sends

Message and OTP send endpoints return `202 Accepted` immediately and are processed on a background queue rather than sent synchronously.

## Errors

Errors use a consistent envelope:

```json
{
  "message": "...",
  "error_code": "...",
  "status": 4xx
}
```
